Changelog

Tracking our journey toward the most stable and modern punishment system. We listen to feedback and change gears when necessary to keep your server running smoothly.

Current Release

v4.3 — The Refocused Update

JANUARY 2026

Following v4.2, we identified areas where the plugin needed better stability under heavy load. This update represents a "change of gears"—reverting experimental systems in favor of a rock-solid core with enhanced database performance and full localization support.

Added
  • MongoDB Parity: NoSQL users now have access to 100% of the features available in SQL.
  • Full Localization: Every GUI title, item name, and lore line can now be translated in your language files.
  • Admin Tools: New built-in data export and import commands for easier backups.
Fixed
  • Database Flooding: Optimized Alt-account menus to reduce database queries by over 75%.
  • Export Crashes: Large data exports now stream to disk instead of using system memory.
  • GUI Fragility: Fixed an issue where changing language could break the punishment menus.
Improved & Changed
  • Void Jail Security: Implemented a "snapshot" saving system to prevent data loss or player location corruption during server crashes.
  • Removed Experimental AI: After reviewing performance cracks in v4.2, we have removed AI-powered moderation to eliminate API latency and ensure punishments remain instant.

v4.2 — The Expansion Update

DECEMBER 2025

This was a major update that pushed the boundaries of traditional moderation by introducing external integrations and experimental auditing features. While ambitious, real-world testing identified performance bottlenecks that led to the refocus found in v4.3.

v4.1 — Web & Proxy Infrastructure

NOVEMBER 2025
  • Introduced the built-in lightweight Web Panel.
  • Added Redis Pub/Sub for sub-millisecond cross-server syncing.
  • Initial Folia native support implementation.
